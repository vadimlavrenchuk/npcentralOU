### 💻 Автоматизация (3 скрипта)

#### 1. **deploy-frontend.ps1**
```powershell
# Автоматический деплой frontend
- Сборка (npm run build)
- Загрузка на сервер (scp)
- Деплой в nginx контейнер
- Проверка
```

#### 2. **deploy-backend.ps1**
```powershell
# Автоматический деплой backend
- Загрузка кода
- Пересборка Docker контейнера
- Проверка логов
```

#### 3. **deploy-all.ps1**
```powershell
# Полный деплой (backend + frontend)
- Backend → Загрузка + Build + Restart
- Frontend → Build + Upload + Deploy
- Финальные проверки
```

### 📦 Интеграция с package.json

Добавлены npm команды:
```json
{
  "scripts": {
    "deploy:frontend": "powershell deploy-frontend.ps1",
    "deploy:backend": "powershell deploy-backend.ps1",
    "deploy:all": "powershell deploy-all.ps1"
  }
}
```

## 🎯 Решенные Проблемы

### Проблема 1: "AI забывает контекст"
**До:** Каждую сессию нужно объяснять заново (5-10 минут)  
**После:** Скопировать COPY_ME_FOR_AI.txt (10 секунд)

### Проблема 2: "AI путает домены"
**До:** Может задеплоить на kontrollitud.ee вместо verifed-est.ee  
**После:** Четко указано в документации и файле COPY_ME_FOR_AI.txt

### Проблема 3: "Не помню команды деплоя"
**До:** Искать в истории команд или гуглить  
**После:** Запустить `npm run deploy:all`

### Проблема 4: "Нет визуального понимания структуры"
**До:** Нужно держать в голове или рисовать каждый раз  
**После:** Открыть INFRASTRUCTURE_DIAGRAM.md

### Проблема 5: "Типичные ошибки повторяются"
**До:** Каждый раз заново ищем решение  
**После:** Все в разделе "Типичные проблемы" DEPLOYMENT_CONFIG.md
